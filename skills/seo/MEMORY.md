# SEO Memory Router

Use this file as the entrypoint to the `memory/` library before giving non-trivial SEO strategy advice.

## How To Use The Library

1. Open [memory/INDEX.md](./memory/INDEX.md) first.
2. Pick the smallest set of notes that fits the task.
3. Prefer memory-backed recommendations over generic SEO advice.
4. When multiple notes overlap, synthesize them instead of repeating them.

## Dedup Rules

- Do not read every note in a cluster by default.
- Start with the canonical note for the topic, then add one supporting note only if the task needs more depth.
- Keep overlap for contrast and detail, not repetition.

## Routing Guide

Use the Ahrefs notes when the task is about:
- keyword research
- competitor analysis
- AI visibility tracking
- AI Overviews and CTR decline
- brand building for SEO
- E-E-A-T audits
- practical workflows and tool-supported operations

Use the Hobo notes when the task is about:
- Google leak interpretation
- NavBoost, Craps, or click behavior
- topicality, site focus, and site radius
- content effort, quality scores, and page quality systems
- leak-aligned on-page, trust, and authority reasoning

Use the source canon notes when the task is about:
- deciding which source type to trust first
- official Google guidance versus industry interpretation
- current rollout and reporting changes
- practitioner experiments and edge-case diagnostics

Use the SEO expert canon when the task is about:
- choosing which expert lens should lead
- building a broader SEO source stack
- deciding between metrics, trust, technical, or AI-search strategy experts
- loading the live expert feed monitor for freshness

Use the Google note when the task is about:
- AI feature eligibility
- structured data alignment
- visible content and markup accuracy
- technical requirements and policy boundaries

Use the industry monitoring note when the task is about:
- current AI-search studies
- Search Console reporting changes
- benchmark tactics the market is testing
- fast-moving platform changes

Use the practitioner note when the task is about:
- AI Mode or AI Overview quirks
- exact rollout dates and change chronology
- quality-system interpretation after updates
- probabilistic retrieval, fan-out, and passage selection

Use the Glenn note when the task is about:
- core updates affecting AI visibility
- AI search traffic reality versus hype
- featured snippets being replaced by AI Overviews
- quality-system losses across multiple Google surfaces

Use the Marie note when the task is about:
- exact AI Mode or AIO rollout dates
- QRG changes affecting AI Overviews
- query fan-out explanations
- whether AI-written content is acceptable under Google's guidance

Use the Patrick note when the task is about:
- AI-search traffic share or conversion quality
- GSC anonymized-query limits
- AI Mode testing against organic ranking
- whether AI-search impact is meaningful yet

Use the Cindy note when the task is about:
- fragment or passage-like visibility
- Fraggles and section-level links
- mobile SERP layout differences
- screenshot-first SERP observation

Use the Lily note when the task is about:
- inaccurate AI Overviews
- Google messaging versus field reality
- trust, brand, and E-E-A-T interpretation
- Gemini grounding and RAG inspection

Use the Brodie note when the task is about:
- AI Mode reporting in GSC
- AIO impression and position behavior
- SERP feature rollout chronology
- measurement quirks on AI surfaces

Use the Mike note when the task is about:
- relevance engineering
- probabilistic retrieval
- semantic competition in AI search
- fan-out and embeddings as strategy

Use the Aleyda note when the task is about:
- international SEO operations
- SERP economics under AI and ads
- market-by-market planning
- curated SEO education resources

Use the Jono note when the task is about:
- technical architecture
- structured data and markup strategy
- rendering and performance
- information architecture

Use the Dejan reverse-engineering pack when the task is about:
- grounding snippets or chunk extraction
- selection rate or SRO
- hidden fan-out or implicit queries
- AI Mode internals and tool orchestration
- machine readability, distillation, or extractability
- reverse engineering why a ranked page is not being selected into AI answers

Use both when the task is about:
- AI search strategy
- zero-click search
- modern SEO measurement
- brand authority and trust
- building systems for a multi-agent SEO workflow

Use Ahrefs + Hobo + Dejan together when the task is about:
- AI visibility diagnostics
- ranking versus selection problems
- citation mining and snippet wins
- designing AI-native content structure and extraction tests

Use the DEJAN note when the task is about:
- grounding snippets and selection rate
- AI search reverse engineering
- citation or snippet diagnostics
- browser or client-code proxy analysis
- legacy technical SEO heuristics applied to AI visibility
